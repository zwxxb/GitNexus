# Move on GitNexus v1.6.6 — Phase 0 Implementation Plan (thin move-flow consumer)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land Move/Aptos support on a fresh GitNexus v1.6.6 base as a **thin consumer of move-flow** — zero raw-source parsing — plus graft the Move MCP tools onto upstream's evolved backend, and produce the move-flow `facts` query spec for Phase 1.

**Architecture:** move-flow MCP is the source of truth. A capability probe selects the richest available query (`facts` if present, else `module_summary` + normalized-signature parsing). A facts→graph mapper writes Move nodes/edges. The fork's fragile raw-source scanning layer (attribute-scanner, enum/resource/friend regex, source-index/store, tree-sitter glue) is **deleted, not ported**. Phase 0 graphs are correct-but-lean; full attribute/acquires/friend/resource/enum fidelity returns in Phase 1 with the facts query.

**Tech Stack:** TypeScript (ESM, Node 22), vitest, tsc, move-flow MCP (JSON-RPC over stdio), Ladybug (lbug) graph store.

**Reference branch (copy-from):** `integrate/upstream-2026-06` → `$REF`
**Base:** `upstream/main` (v1.6.6 @ `2cf7e7fa`)
**Merge-base:** `baf3f9e3` → `$BASE`

**Module keep/delete decision (the 25 fork `core/move/` modules):**
- **KEEP & adapt:** `mcp-client.ts` (add capability probe), `move-ingest.ts` (rewrite thin), `compiler-facts.ts` (add facts types), `move-toml.ts` (manifest = config, not source), `signature-parser.ts` (parses *normalized* compiler signatures — robust), `symbol-id.ts`, `graph-writer.ts`, `port-extractor.ts`, `quality-phase.ts`, `verification-enrichment.ts`, `entry-points.ts` (operates on flags, not source), `consistency.ts`, `index.ts`.
- **DELETE (do not copy):** `attribute-scanner.ts`, `enum-variants.ts`, `resource-ops.ts`, `friend-resolver.ts`, `source-store.ts`, `source-utils.ts`, `source-index.ts`, `language-provider.ts`, `type-extractor.ts`, `type-visibility.ts`, `named-bindings.ts`, `import-resolver.ts`.

**Conventions:** `tsc` = `npx tsc --noEmit` in the named package dir. `vitest <pat>` = `cd gitnexus && npx vitest run <pat>`. "Inspect fork diff" = `git diff $BASE..$REF -- <file>`.

- [ ] **Step 4: Implement the capability probe**

In `gitnexus/src/core/move/mcp-client.ts`, add:
```ts
export interface MoveFlowCapabilities {
  hasFactsQuery: boolean;       // move_package_facts (or query:"facts") available
  hasModuleSummary: boolean;    // move_package_query module_summary available
}

/** Derive capabilities from a move-flow tools/list response (tool names). */
export function detectMoveFlowCapabilities(toolNames: readonly string[]): MoveFlowCapabilities {
  const names = new Set(toolNames);
  return {
    hasFactsQuery: names.has('move_package_facts'),
    hasModuleSummary: names.has('move_package_query'),
  };
}
```
Then add a `capabilities()` method that calls `tools/list` over the existing JSON-RPC channel,
caches the result, and returns `detectMoveFlowCapabilities(...)`. Add a `facts(packagePath)`
method that calls `move_package_facts` (or `move_package_query` with `query:'facts'`) — guarded
so it is only invoked when `hasFactsQuery` is true.
